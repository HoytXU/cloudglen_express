A procedural landscape built from noise, layers, and masks

1

A continuous random line

Begin with two numbers, a and b. To find a value between them, move a fraction t of the way from the first to the second. This is the same idea as finding a point partway along a ruler.

v(t) = (1 − t)a + tb

Random values at consecutive grid points would form a jagged line if joined directly. Replacing t with a smooth fade s(t) rounds each join. Perlin introduced the quintic fade used here [2].

s(q) = 6q5 − 15q4 + 10q3
n(p) = lerp(lerp(a,b,ux), lerp(c,d,ux), uy) 

vec2 f = fract(p);
vec2 u = f*f*f*(f*(f*6.0-15.0)+10.0);

return mix(mix(a, b, u.x),
           mix(c, d, u.x), u.y);

The full shader works on a square rather than a line. It blends the lower two corners, blends the upper two corners, then blends those two answers vertically. The figure is one horizontal slice through this two-dimensional field.

p is the pixel position. fract(p) keeps only its position inside the current grid square.

a, b, c, d are the four random corner values. The final scene reads them from a blue-noise texture; reference [4] explains why blue-noise samples are spatially well distributed.

The fade has flat ends, so adjacent squares meet without a visible corner. It changes the route between the values, not the values themselves.

2

One cloud boundary

One smooth noise line makes broad hills but little detail. Imagine drawing a large wavy coastline, then adding smaller wiggles with a lighter pencil. The shader does the same by adding scaled copies of noise.

cloud = broad shape + smaller detail + fine detail

Each copy is called an octave. This octave construction is commonly called fBm in procedural graphics [3].

C(p) = ΣK−1k=0 gkn(2kp)ΣK−1k=0 gk 

float sum = 0.0, weight = 0.0;
float amplitude = 1.0;
for (int k = 0; k < K; ++k) {
  sum += amplitude * noise(p);
  weight += amplitude;
  p *= 2.0;
  amplitude *= 0.7;
}
return sum / weight;

Each octave doubles spatial frequency and multiplies amplitude by g = 0.7. Division by the accumulated weight keeps the range stable. The comparison uses three octaves; the final cloud shader uses eight. Smoke uses the same construction with a slower decay, g = 0.9, which retains more fine-scale variation.

The summation symbol in the panel is shorthand for repeatedly adding terms. For the three-scale figure it expands to:

C(p) = [n(p) + 0.7n(2p) + 0.49n(4p)] / 2.19

2p makes features half as wide; 4p makes them one quarter as wide. The weights 0.7 and 0.49 make the smaller features weaker.

The denominator 2.19 is simply 1 + 0.7 + 0.49. Dividing by it prevents the image from becoming brighter or taller just because more octaves were added.

3

Layers and parallax

Look through a train window: a nearby fence sweeps past quickly while distant hills barely move. This is parallax. The shader imitates it by dividing time by an artificial distance d.

horizontal shift = time / distance

The same cloud function is then reused at different heights, sizes, phases, and distances. Painting slow pale layers first and fast dark layers last produces a simple sense of depth.

hi(x,t) = mi + Ai [C(λi(x + t/di) + φi) − ½] 

float q = frequency
        * (x + time / distance)
        + phase;
return middle + spread
        * (cloudNoise(vec2(q, 1.7)) - 0.5);

The symbols correspond directly to the code: m is middle, A is spread, and d is distance. The frequency λ controls the horizontal feature size, while φ gives each layer a different slice of the noise field. Since velocity is proportional to 1/d, a larger distance produces slower motion.

Read the equation from inside outward: move the sample by t/d; use λ to choose the feature size; use φ to select a different part of the noise; then evaluate the cloud.

Because cloud noise lies roughly between 0 and 1, subtracting ½ centres it around zero. A controls how far it moves up and down, and m lifts the complete line to its intended height.

For example, a layer with d = 20 moves one tenth as fast as a layer with d = 2. This relative motion is the depth cue.

4

Train and bridge masks

A mask is a stencil: 1 means “paint this pixel” and 0 means “leave the existing colour.” A wheel begins with the familiar definition of a circle—every point closer than radius r to its centre c.

inside the wheel when distance(pixel, centre) < radius

Rectangular masks form the body and chimney. Repeating the horizontal coordinate produces carriages and bridge spans without listing every copy separately.

Mwheel = 1 − smoothstep(r, r + ε, ‖p − c‖)
Mtrain = max(Mbody, Mchimney, Mwheels) 

float wheel = 1.0 - smoothstep(
  radius, radius + 0.004,
  length(uv - center)
);
float train = max(max(body, chimney), wheels);

The wheel mask is one inside the radius and zero outside it; the narrow ε interval softens its edge. Taking the maximum is the union of the component masks. The compact study combines them into one colour, while the final shader retains separate masks for the wagon, roof, locomotive, chimney, and wheels so those parts can use different tones. It also uses a hard step test on squared distance for its much smaller final-render wheels.

The distance ‖p−c‖ asks how far the current pixel is from a wheel centre. Pixels nearer than radius r receive a mask value of 1; pixels outside r + ε receive 0.

smoothstep changes gradually from 0 to 1 across the tiny interval ε. Subtracting it from 1 makes the inside opaque and the outside transparent while keeping the edge smooth.

max performs a union: if any component contains the pixel, the complete train mask contains it.

5

A widening, irregular smoke plume

Imagine drawing a thick ribbon around a pencil line. The line gives the smoke its path; a width value says how far pixels may lie from that path and still belong to the plume.

smoke pixel when |pixel height − centre height| < width

The shader bends the centreline upward with a square root, changes the width with distance from the chimney, and adds fBm so the ribbon does not look mechanically perfect.

x = uchimney − ux
c(x,t) = 0.16√x + 0.12 − 0.4N(u,t)
w(x) = 0.8xe−10x
Msmoke = [|uy−c(x,t)| < w(x)]

The horizontal distance x is zero at the chimney and increases behind the train. The square root bends the plume upward quickly near the chimney and more gradually farther away.

N denotes fbm2(...) − 0.55: the eight-octave smoke noise with gain 0.9, shifted so it fluctuates around zero. Subtracting it from the centreline makes the plume wander rather than follow a perfect curve.

In w(x) = 0.8xe−10x, the factor x first widens the plume, while the exponential factor eventually shrinks it. Together they reach maximum width at x = 0.1.

6

Final expression

Think of the scene as transparent sheets placed on a desk. The sheet placed last hides earlier sheets wherever it is opaque. In GLSL, one such placement is a colour blend controlled by a mask.

new colour = mix(old colour, layer colour, layer mask)

The production shader draws background clouds, train, smoke, bridge, and foreground clouds in that order, following the source composition [1].

F(u,t) = V(u) [Lbg ⊕ Mtrain ⊕ Ssmoke ⊕ Mbridge ⊕ Lfg]

V(u) = ½ + ½[16uxuy(1−ux)(1−uy)]0.2

The operator ⊕ means “draw over,” from left to right. The vignette equals 1 at the image centre and falls to 0.5 at its edges.

The ⊕ symbols mean the same ordered “draw over” operation as mix. Foreground clouds can cover the bridge, and the bridge can cover smoke where their masks overlap.

The second pass multiplies the finished scene by V. At the image centre, both coordinates are ½ and V = 1, so the colour is unchanged. At any edge, V = 0.5, so the colour is half as bright.
