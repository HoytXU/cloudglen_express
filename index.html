Cloudglen Express

A procedural landscape built from noise, layers, and masks

Press F for fullscreen

From one function to a complete scene

The image is generated without photographs, meshes, or a 3D camera. A smooth random function defines one cloud edge. Weighted copies add detail, repeated layers create depth, and simple masks draw the train.

1

A continuous random line

Value noise interpolates between four random values at the corners of each two-dimensional lattice cell. The figure shows a horizontal slice through that field. A quintic fade curve makes the value, slope, and curvature meet smoothly at every cell boundary.

s(q) = 6q5 − 15q4 + 10q3
n(p) = lerp(lerp(a,b,ux), lerp(c,d,ux), uy) 

vec2 f = fract(p);
vec2 u = f*f*f*(f*(f*6.0-15.0)+10.0);

return mix(mix(a, b, u.x),
           mix(c, d, u.x), u.y);

Here f is the position inside one cell, u = s(f) is the component-wise fade, and a–d are its corner values. The study hashes the lattice coordinates; the final shader reads those values from the blue-noise texture. Both then perform the same interpolation.

2

One cloud boundary

Several noise scales are added together. Large features carry the silhouette; faster and weaker signals supply progressively finer detail. This weighted, normalized sum is commonly called fractal Brownian motion (fBm) in procedural graphics.

C(p) = ΣK−1k=0 gkn(2kp)ΣK−1k=0 gk 

float sum = 0.0, weight = 0.0;
float amplitude = 1.0;
for (int k = 0; k < K; ++k) {
  sum += amplitude * noise(p);
  weight += amplitude;
  p *= 2.0;
  amplitude *= 0.7;
}
return sum / weight;

Each octave doubles spatial frequency and multiplies amplitude by g = 0.7. Division by the accumulated weight keeps the range stable. The comparison uses three octaves; the final cloud shader uses eight. Smoke uses the same construction with a slower decay, g = 0.9, which retains more fine-scale variation.

3

Layers and parallax

The same cloud function is evaluated at several heights and phases. Distant layers move slowly; nearby layers move quickly and cover the layers behind them.

hi(x,t) = mi + Ai [C(λi(x + t/di) + φi) − ½] 

float q = frequency
        * (x + time / distance)
        + phase;
return middle + spread
        * (cloudNoise(vec2(q, 1.7)) - 0.5);

The symbols correspond directly to the code: m is middle, A is spread, and d is distance. The frequency λ controls the horizontal feature size, while φ gives each layer a different slice of the noise field. Since velocity is proportional to 1/d, a larger distance produces slower motion.

4

Train and bridge masks

Rectangles define the train body, distance tests define the wheels, and repeated coordinates define the bridge. Their hard silhouette remains legible against the continuous cloud boundaries.

Mwheel = 1 − smoothstep(r, r + ε, ‖p − c‖)
Mtrain = max(Mbody, Mchimney, Mwheels) 

float wheel = 1.0 - smoothstep(
  radius, radius + 0.004,
  length(uv - center)
);
float train = max(max(body, chimney), wheels);

The wheel mask is one inside the radius and zero outside it; the narrow ε interval softens its edge. Taking the maximum is the union of the component masks. The compact study combines them into one colour, while the final shader retains separate masks for the wagon, roof, locomotive, chimney, and wheels so those parts can use different tones. It also uses a hard step test on squared distance for its much smaller final-render wheels.

5

Final expression

Each pixel is formed by drawing the distant layers first, then the train, smoke, bridge, and foreground clouds. This order matches the sequence of mix operations in the production shader.

F(u,t) = V(u) [Lbg ⊕ Mtrain ⊕ Ssmoke ⊕ Mbridge ⊕ Lfg]

V(u) = ½ + ½[16uxuy(1−ux)(1−uy)]0.2

The operator ⊕ means “draw over,” from left to right. The vignette equals 1 at the image centre and falls to 0.5 at its edges.
