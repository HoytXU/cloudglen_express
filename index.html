Interpolation

Let a and b be two real numbers. Let τ (Greek “tau”) be a parameter between 0 and 1 that measures progress from a to b. Define the linear interpolation operator L(a,b;τ) = (1−τ)a + τb. The interpolated value may therefore be written x(τ)=L(a,b;τ). Thus L(a,b;0)=a and L(a,b;1)=b; intermediate values lie on the line segment joining a and b. GLSL implements L as mix.

In two dimensions, τ1 measures horizontal progress and τ2 measures vertical progress.

Value noise with Perlin’s fade

The shader divides the image plane into square cells and assigns a random scalar to each corner. If adjacent values are joined by straight lines, each cell has a constant slope, but the slope usually changes abruptly at the next grid point. The visible result is a corner. The fade function replaces that constant-speed transition with one that starts and ends flat.

Perlin’s quintic fade [2] preserves the endpoints: S(0)=0 and S(1)=1. Its first and second derivatives are also zero at both ends. In geometric terms, every cell reaches a shared boundary with zero slope and zero curvature, so neighbouring cells agree in value, slope, and curvature.

A point p has global image coordinates. The operator fract discards the whole-number cell index and retains only the position inside that cell: ξ=fract(p).

The subscripts in vij identify a corner: i=0 or 1 selects left or right, and j=0 or 1 selects bottom or top. Applying S to the two local coordinates gives w=(S(ξ1),S(ξ2)). These are interpolation weights, not a second spatial position. The symbols b0 and b1 below denote the horizontal blends along the bottom and top rows. Their vertical blend is the final value N(p). The production shader reads its corner samples from a blue-noise texture [4]; the heatmap uses that same texture and the same production N function.

The visual below samples the square region [0,4]². Each image position represents a point p=(x,y), and its grayscale brightness represents the scalar value N(p): black denotes 0 and white denotes 1. The thin grid marks the sixteen unit cells. The marked point illustrates one evaluation of the field. Grayscale is only a numerical legend here; it is not the cloud colour used by the scene.

ξ = fract(p),   w = (S(ξ1), S(ξ2))
b0 = L(v00,v10;w1)
b1 = L(v01,v11;w1)
N(p) = L(b0,b1;w2) 

vec2 xi = fract(p);
vec2 w = xi*xi*xi
  * (xi*(xi*6.0-15.0)+10.0);
float bottom = mix(v00, v10, w.x);
float top = mix(v01, v11, w.x);
return mix(bottom, top, w.y);

Octaves

Let N:ℝ²→[0,1] be the value-noise field from Section 1. A single evaluation scale contains broad variation but little fine structure. Additional copies of N, evaluated at successively higher spatial frequencies and lower amplitudes, add smaller features. Write F3,0.7 for the normalized combination of three such scales; the subscript records three scales and an amplitude multiplier of 0.7.

Start with only two octaves. The first field supplies broad variation; the second evaluates the same function at twice the spatial frequency and contributes with relative weight g. Division by 1+g keeps the result in the original numerical range.

F2,g(p) = [N(p) + gN(2p)]/(1+g)

F3,0.7(p) = [0.70N(20p) + 0.71N(21p) + 0.72N(22p)] / [0.70+0.71+0.72] = [N(p) + 0.7N(2p) + 0.49N(4p)] / 2.19.

This is the general formula with K=3 and g=0.7 substituted into it. The sum contains exactly the indices k=0,1,2: 0.7²=0.49, 2²=4, and 1+0.7+0.49=2.19. Thus the two displayed forms are the same expression, not different definitions.

Each term is called an octave. More generally, K denotes a positive integer number of octaves, and g denotes an amplitude gain strictly between 0 and 1. The index k counts the octaves from 0 to K−1, and Σ means “sum these terms over k.” The resulting normalized octave sum is commonly called fractal Brownian motion (fBm) in procedural graphics [3].

In the general expression below, multiplying p by 2k makes octave k vary 2k times as rapidly in space, while gk reduces its contribution. Division by Σgk makes the weights sum to 1, so FK,g(p) remains between 0 and 1 whenever N(p) does.

The loop mirrors the formula: p *= 2 produces 2kp, amplitude *= gain produces gk, and sum / weight performs the normalization.

The six heatmaps below use the same base region p∈[0,4]². The top row isolates the first three raw octave fields. The bottom row accumulates two, three, and eight octaves. The first top panel is exactly Section 1, and the final bottom panel is the eight-octave field used by the production clouds. Eight octaves retain substantially finer variation than two or three; each additional octave requires another noise evaluation, so this detail increases shader cost approximately linearly with K.

FK,g(p) = ΣK−1k=0 gkN(2kp)ΣK−1k=0 gk 

float sum = 0.0, weight = 0.0;
float amplitude = 1.0;
float gain = 0.7;
for (int k = 0; k < K; ++k) {
  sum += amplitude * noise(p);
  weight += amplitude;
  p *= 2.0;
  amplitude *= gain;
}
return sum / weight;

Parallax

Section 2 displays F8,0.7 as grayscale so that its numerical values are visible. It first converts them into the boundary of a filled region.

Begin with a one-dimensional slice f(x)=F8,0.7((x,0)). Let m be a baseline height and A>0 a vertical amplitude. Subtracting ½ centres the noise around zero: values above ½ raise the boundary and values below ½ lower it. Filling every point below that boundary produces a cloud silhouette.

h(x) = m + A[f(x)−½]
M(x,y) = 𝟙{y<h(x)}

Parallax is then introduced by translating the input of the same noise field. The noise pattern is unchanged; only the position at which it is sampled changes with time and apparent distance.

Let p=(p1,p2) be the height-normalized scene coordinate, and let t denote scaled animation time. The subscript i identifies a cloud layer. Each layer has a positive apparent distance di and a horizontal phase φi.

Δxi(t) = −t/di,   dΔxi/dt = −1/di

The shader samples at p1+t/di; therefore a fixed feature appears at p1= constant−t/di and moves left. Its speed magnitude is 1/di, so distant layers move more slowly.

The production version replaces the introductory horizontal slice by a full two-dimensional lookup. At every pixel, it evaluates the noise, converts that value into a local threshold, and tests whether p2 lies below it. Define the exact lookup position qi, field value zi, and binary layer mask Mi as shown below. The parameters mi and Ai are the layer’s middle height and displacement amplitude. Both coordinates of p enter the production noise field.

The four panels isolate the outer masks of actual background layers c12, c11, c10, and c9 from the final shader. White is outside the mask and black is inside. Their distances are 10, 15, 20, and 25, so the animation directly shows the decreasing speed 1/di. The final scene evaluates the same masks in order and assigns each one its production colours.

qi(p,t) = p + (t/di + φi,0)
zi(p,t) = F8,0.7(qi(p,t))
Mi(p,t) =
𝟙{p2 < mi + Ai[zi(p,t)−½]} 

vec2 q = p + vec2(
  time / distance + phase, 0.0
);
float z = fbm(q, 8);
float mask = 1.0
  - step(middle + amplitude*(z-0.5), p.y);

Masks

A spatial mask M assigns every point p in the two-dimensional image plane a number from 0 to 1. The values 0 and 1 mean exclusion and inclusion. The production train is assembled from rectangle masks, repeated wagon masks, and hard circular wheel masks. The symbol 𝟙 denotes an indicator: it returns 1 when its subscript condition is true and 0 otherwise.

Begin in one dimension. An interval mask selects the points between a and b. A rectangle applies the same test independently to horizontal and vertical coordinates; a disk replaces those coordinate tests with a distance test.

M[a,b](x) = 𝟙{a<x<b}

Mwheel(p) = 𝟙{‖p−c‖² < r²}

The operation fract(9p) repeats local coordinates every 1/9 scene units, allowing one wagon and wheel pattern to generate the complete train. The combined train mask is the pointwise maximum of its component masks, which is their set-theoretic union. These analytic tests avoid mesh or image assets and remain sharply defined at any output resolution.

The bridge function in the source shader computes a keep mask KB: it equals 1 where the existing scene colour is preserved and 0 where bridge colour is inserted. Therefore the bridge inclusion mask is MB=1−KB.

The left panel enlarges the train mask and the right shows the bridge mask. Black denotes 0 and white denotes 1. In the maximum below, every component mask is evaluated at the same point p; the repeated arguments are omitted.

Mtrain(p) = max(
Mwagon,Mjoin,Mroof,
Mlocomotive,Mlocomotive roof,
Mchimney,Mwheels)
MB(p,t) = 1−KB(p,t) 

float wheel = 1.0
  - step(r*r, dot(p-c, p-c));
float train = max(body,
  max(chimney, wheel));
float bridge = 1.0 - bridgeKeep;

Smoke

Start with an ordinary horizontal strip. Let p=(x,y) be a point in the scene, let c be the height of the strip’s centre, and let w>0 be its half-width. The vertical distance from p to the centre is |y−c|. Therefore the point is inside the strip precisely when |y−c|<w.

D(p) = |y−c|−w
D<0: inside,   D=0: boundary,   D>0: outside

Subtracting w from the distance creates a signed boundary value D. A negative result means that the point is less than one half-width from the centre and is therefore inside. Smoke is obtained by allowing the centre c and width w to change with position and time.

The chimney is at x0=0.49. Smoke exists to its left, so define s=x0−x≥0. The number s is simply the horizontal distance travelled from the chimney: s=0 at the source and increases farther along the plume.

Let R(p,t) be the eight-octave field from Section 2, sampled at a horizontally moving position. This is the grayscale field in the left panel. The centre height starts with the smooth curve 0.16√s+0.12; the additional term −0.4[R−0.55] moves that curve up and down according to the noise.

The width is w(s)=0.8se−10s. It begins at zero, widens until s=0.1, and then narrows. The factor s causes the initial growth, while the exponential factor e−10s eventually causes the decay. Substituting this width and the noisy centre into the same strip test produces the plume. The condition D<0 selects all smoke; the stricter condition D<−0.02 selects points farther inside and forms the inner region.

Thus the left panel is not smoke by itself. It supplies the small vertical changes in the centre curve. The right panel shows the result after the centre, width, and inside–outside test have been combined.

F8,0.9(p+(0.16t+3.5,0))

0.16√s+0.12−0.4[R(p,t)−0.55]
w(s) = 0.8se−10s
D(p,t) = |y−c(p,t)|−w(s)
Mouter = 𝟙{D<0},   Minner = 𝟙{D<−0.02}

  0.16*time + 3.5, 0.0), 8);
float center = 0.16*sqrt(s) + 0.12
  - 0.4*(R - 0.55);
float width = 0.8*s*exp(-10.0*s);
float D = abs(p.y - center) - width;
float outer = 1.0 - step(0.0, D);
float inner = 1.0 - step(-0.02, D);

Composition

For viewport width W>0 and height H>0, define normalized display coordinates by u=(xfrag,1/W, xfrag,2/H)∈[0,1]². Unlike the height-normalized coordinate p, this definition scales each coordinate by the corresponding viewport dimension.

For colours a,b ∈[0,1]³ and mask value m∈[0,1], define the compositing operator Blend by component-wise linear interpolation.

Blend(a,b;m) = (1−m)a + mb

The mask m is a weight. At m=0 the result is entirely a; at m=1 it is entirely b; and at m=½ both colours contribute equally. When two opaque masks overlap, the operation applied later determines the visible colour in the overlap.

Sequential application of Blend is order-dependent: a later layer replaces an earlier colour wherever its mask equals 1. The production order follows the source shader [1].

Let J be the number of ordered paint operations in the scene pass. Operation j has source colour sj and mask mj. These operations include the separately coloured train components, the two smoke regions, the inverse bridge keep mask, and the foreground cloud opacity. The intermediate colour after operation j is cj.

Every colour and mask below is evaluated at the same pixel and time; those arguments are omitted for legibility. The renderer applies this recurrence in source order: each mix is one instance of Blend.

After compositing, V:[0,1]²→[0.5,1] applies the vignette. It equals 1 at u=(½,½) and 0.5 on every image boundary, so the centre is unchanged while the edges are multiplied by one half. This frames the central train and cloud layers without changing their geometry or compositing order.

c0 = cbg
cj = Blend(cj−1,sj;mj),   j=1,…,J
cscene = cJ
cout(u,t) = V(u)cscene(u,t)

V(u) = ½ + ½[16u1u2(1−u1)(1−u2)]0.2

The left panel displays V; the right multiplies a reference grid by V. The final showcase applies the same factor to the completed scene.

V(u) cscene(u,t)

  * (1.0-uv.x) * (1.0-uv.y);

vec3 outputColor = V * sceneColor;
